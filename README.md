# SOFTWARE DE DELIVERY

# REQUISITOS FUNCIONAIS
1. O software deve ter cadastro do usuário 
2. O software deve ter o login
3. O software deve aceitar o cancelamento do pedido
4. O software deve ter opção de tele entrega ou retirada
5. O software deve aceitar várias formas de pagamento
6. O software deve buscar localização do usuário 
7. O software deve ter a opção de mostrar o andamento da comida
8. O software deve aceitar a escolha de um ou mais pratos 
9. O software deve mostrar quando o estabelecimento estiver fechado
10. O software deve calcular automaticamente o valor total do pedido
11. O software deve permitir a recuperação de senha
12. O software deve permitir adicionar e remover itens do carrinho
13. O software deve permitir que o usuário informe ou altere o endereço de entrega
14. O software deve permitir a realização de pedidos
15. O software deve confirmar o pagamento antes de finalizar o pedido
16. O software deve manter um histórico de pedidos do usuário
17. O software deve permitir avaliação de pedidos e estabelecimentos
18. O software deve permitir que o usuário edite seus dados cadastrais
19. O software deve permitir a visualização de estabelecimentos disponíveis
20. O software deve exibir o cardápio completo de cada estabelecimento
21. O software deve permitir a aplicação de cupons de desconto
22. O software deve calcular automaticamente taxas de entrega
23. O software deve exibir o tempo estimado de entrega 
24. O software deve permitir favoritar estabelecimentos e pratos
25. O software deve permitir a busca de estabelecimentos por nome, tipo de comida ou localização
26. O software deve permitir comunicação entre cliente e restaurante
27. O software deve gerar comprovante do pedido
28. O software deve permitir agendamento de pedidos para horário futuro
29. O software deve permitir denunciar problemas com pedidos
30. O software deve exibir a distância entre usuário e estabelecimento
31. O software deve permitir que estabelecimentos definam horário de funcionamento
31. O software deve permitir configurar valor mínimo de pedido
31. O software deve exibir detalhes completos dos produtos (descrição, preço, imagem)
32. O software deve permitir que o usuário visualize promoções ativas
33. O software deve permitir pagamento online e na entrega


# REQUISITO NÃO FUNCIONAIS
1. O software deve ser atualizado de forma rápida o andamento da comida
2. O software deve proteger os dados dos seus usuários 
3. O sistema deve carregar as telas principais de forma rapida
4. A interface deve ser intuitiva e fácil de usar para diferentes idades
5. O sistema deve suportar aumento no número de usuários e pedidos sem perda de desempenho
6. O sistema deve estar disponivel a todo o momento 
7. O sistema deve estar disponível em dispositivos móveis e desktop
